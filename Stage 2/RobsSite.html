Why am I seeing this?

You are on a browser (hopefully, or else then I have no answer for you). You told your browser to go to this page, and so your computer, since it's dumb and has no mind of its own (yet), went ahead and texted its friend Web and said "Hey buddy, can you send me the files that my owner requested?"

Web said, "no problem, just give me a few nanoseconds," then retreived the files from a server via something called HTTP and sent them to your computer. Then your browser, since it's also dumb and has no mind of its own, interpeted the files that came through and displayed them on your screen.

Ok, but how did you make it?

I used a language called HTML, which stands for Hypertext Markup Language. HTML is made up of:

  • Text content - which is what you are seeing now
  • Markup - which is what this page looks like
  • References to other documents - like the logo at the top of the screen/li>
  • Links to other pages - like this link to a puppy video

How did you make it look like this?

Everything you see on the screen is part of an HTML element. You add these elements to your page using things called tags, and you can apply attributes, or styles, to those elements. So, for example, I added a link to the puppy video using a link tag (which looks like this: <a>), and I wrote in my code that all link elements should be my favorite shade of red and have bold font.

Oh, so how do your write HTML?

To start, you can learn one easy thing, which is that every HTML document has the same basic structure:

  • HTML <html> - which tells the computer that you are writing a document using HTML
  • Head <head> - where you usually put the styling of your page
  • Body <body> - where pretty much everything else goes

So now you're saying it is easy?

Well, to start is easy. But then what? You need to know some more essentials. For instance, there are other languages besides HTML. One language that's used for pretty much every web site is CSS. HTML controls the "structure" of a webpage, while CSS controls the "style" of the page.

You can put your styling directly in to your HTML, but CSS is essential to code writers because it helps them avoid a lot of repetition. I already gave an example of this above. To avoid having to tell my HTML every time that I wanted my links red, I instead only had to type it once in my CSS.

Tell me more about HTML and CSS

As I said, the <head> section is usually where you put styling, so if you have a CSS file you can simply put a link to it in the <head> section of your HTML and voila, your HTML will now listen to what your CSS tells it to do.

Programmers use a "tree-like structure" when coding their sites. We call it that because HTML elements can have other elements inside of them, kind of like a family tree. And just like a family, the "child" elements can "inherit" attributes from a "parent", which, again, helps programmers avoid repetition.

But what about these boxes?

You're catching on already! You can see that there are boxes on this page. But even when you can't see boxes on the screen, know that the page was still built using boxes. Every page has boxes! Every box on your screen is built of four elements:

  • Content - in the middle, contains the text and images you see on the screen. The other layers are like protection to the content.
  • Padding - this clears an area around the content. Like how there is a little space between the red text and the gray side of your screen.
  • Border - this goes around the content and padding. These boxes have a gray border on the bottom.
  • Margin - the outermost layer, it provides spaces between the boxes on the page.

How do you get the boxes where you want them to go?

Box sizing and positioning is one of the hardest parts of building a web page. You can set the size of your box with pixels, or in percentages. I used percentages on this page.

Sweet! Now I want to get serious

To begin "serious" programming, you must first understand some Computer Science. The essense of Computer Science is taking a big problem, and then breaking it down into smaller pieces, each of which can be described in a series of precise steps. Those steps then get executed by a computer.

Unlike a toaster, which does basically one thing, a computer can do anything, if you write a program for it.

So how can you write a program for a computer?

It would be easy to just type in the language you know already to tell a computer what to do. The problem with that is languages can be ambiguous, and computers need very precise instructions to execute programs correctly. This is why we have invented programming languages, like Python, which have precise rules that computers can respond to.

Python is an interpreter. In other words, instead of writing your program directly on your computer, Python takes the code you've written, and translates it into a language that the computer can read.

Python? Sounds scary

It's not so bad. A Python program can be as simple as easy arithmetic. It can be broken down into the syntax: Expression -> Expression Operator Expression, kind of like how a sentence can be broken down into Subject Verb Object. So, an example of a very simple Python program would be 'print 2 + 2'.

This seems like a lot to remember

While it's nice to know the language syntax of Python and other languages, what is most important to remember is that programming requires three important concepts:

  1. Procedural thinking
  2. Technological empathy for how computers work
  3. Abstract thinking, like thinking of a Python program, a Python interpreter, and a web browser as different versions of the same thing (a computer program)

Python sounds tedious

It could be, but what can help you avoid repition is using variables. A variable is a name for something. So if you are going to use the number '19082098' a bunch of times in your code you could set a variable:

number = 19082098

This way, instead of typing the whole number every time you could type the word 'number' in your code. This also makes your code a lot more readable to both yourself, and to others who may see your code and/or need to make changes to it.

Equal sign? Like algebra?

It's actually different than algebra; the equal sign does not make an equation. Instead, "=" essentially means assign a value to a name. So "number = 19082098" really means "assign the value 19082098 to the name 'number'"

What else is in Python?

Another important concept is String, which is a sequence of characters inside quote signs. So when I say...

print "Hello"

... the Python interperet will return: Hello

And other important concepts are String Indexing, which allows you to return a value based on a position in the string, and the Find method, which allows you to find the position of a value within the string.

K, I'm ready to code now

Hold on there buddy, we need to talk about Functions. Essentially, a function is a way to tell your code to go through a certain procedure repeatedly. So, I could say, everytime I type "sumthesenumbers" you should take the two numbers and sum them. This helps me avoid repition since I then won't have to type the procedure every time.

A function takes inputs, puts them through a procedure, and then gives you outputs. If my function says to sum two numbers, then when I input the two numbers 3 and 4, it wil give me an output of 7.

That is pretty cool. Tell me more.

One essential thing to remember is to always include a "return" statement in your function. If you don't, the computer will not actually return a result. It will just say "None" and the user of your website will be like what, why???

What are some other functions?

One really important function is if. That allows you to test whether a condition is true, and return a certain output if it is, and another output if it isn't. One of the most common uses of "if" is to do comparison.

The amazing thing about "if" is that with procedures, comparison, ifs, and simple arithmetic, you can write every possible computer program.

Great, so I guess we're done!

Again, no. While you can technically write any program, there are a lot more concepts to learn that will make programming easier. Two important concepts that get rid of a lot of repition are For and While Loops. They both allow you to check a condition or apply a procedure to elements in a set, instead of have to write it over and over again.

Fine, I'm not ready. What else should I know?

Another important concept: lists. Lists are different than strings in that strings have to be a sequence of characters, while lists can contain anything. An example of a list: p = ['R', 'o', 'b', 's', 'S', 'i', 't', 'e']. Lists are great because they are what we call a mutable object, meaning we can give them mutations, which sounds scary but really just means we can change the values within the list.

Mutations??? What else can you do?

Here are a few important list operations which we can use to change or act upon lists:

  • append (adds new elements to the end of a list)
  • + (combines two lists together)
  • len (short for length; finds the number of element in a list)
  • index (finds the first position of a value in a list)
  • in (sees whether a value is in a list and gives an output of 'True' or 'False')
